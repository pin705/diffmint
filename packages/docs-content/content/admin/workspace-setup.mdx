# Workspace Setup

## Admin rollout checklist

1. Create the workspace.
2. Configure provider strategy.
3. Publish the first policy version.
4. Confirm billing and quota defaults.
5. Invite members.
6. Share the CLI and VS Code install guides.

The dashboard is the control plane for this flow. Day-to-day reviews should still happen in the CLI or editor.

After rollout, admins can export synced review history and audit trail CSVs directly from the
dashboard for governance reviews, support handoffs, or compliance workflows.
