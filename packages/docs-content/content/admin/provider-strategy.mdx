# Provider Strategy

Every workspace should choose one provider operating model before broad rollout. Keep the decision
explicit so users know who owns credentials, model choice, and failure handling.

## Operating models

### Managed provider

Choose this when the platform team wants one centrally managed experience with the least user-side
setup burden.

### Bring your own key

Choose this when security, procurement, or business constraints require each workspace to own its
own provider credentials and model access.

## Recommended defaults

- one primary provider
- one fallback strategy
- one default model pinned per workspace
- workspace-level rate and spend ownership
- explicit secret rotation ownership
- model allow-lists defined before rollout, not after incidents

## Decision checklist

Confirm these before inviting the full team:

1. Who owns provider credentials?
2. Which models are approved by default?
3. What happens when the preferred provider is unavailable?
4. Who rotates secrets and audits access?
5. Which choices are user-configurable versus workspace-wide?
