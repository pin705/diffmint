# Docker Development

Use Docker Compose when you want the web app and Postgres to boot together with hot reload.

## Start the stack

```bash
pnpm install
pnpm docker:dev
```

This brings up:

- `web` on `http://localhost:3000`
- `postgres` on `localhost:5432`

The web container now runs database migrations and the idempotent Diffmint seed before starting the
Next.js dev server, so the control plane is ready on first boot.

The Docker stack also probes `http://127.0.0.1:3000/api/health/ready`, which means container health
now reflects both the web runtime and the control-plane readiness checks.

Make sure Docker Desktop or your local Docker daemon is running before you start the stack.

## Why this setup

- `pnpm` is the workspace package manager
- the compose stack mounts the repo for live edits
- polling is enabled so file watching stays reliable inside Docker
- the Postgres service matches the `DATABASE_URL` used by the web app
- the web service exposes live and readiness probes for deployment platforms and local diagnostics

When you promote this setup toward production, enable `DIFFMINT_REQUIRE_PERSISTENCE=true` so the
control plane fails fast if Postgres is unavailable instead of serving fallback memory state.

## Useful commands

```bash
pnpm docker:dev:logs
pnpm docker:dev:down
pnpm docker:dev:reset
pnpm db:generate
pnpm db:migrate
pnpm db:seed
```
