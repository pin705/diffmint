# Install the VS Code Extension

The VS Code extension keeps the review flow inside the editor with native panels, command palette
actions, and context-aware navigation. By default it uses `dm` when that binary is already
available, and falls back to an auto-installed managed CLI when it is not.

## Before you install

- Decide whether the extension should point at production, self-hosted, or a local control plane
- If your team manages a custom CLI build, know the path you want to pin in `diffmint.cliPath`

## Setup flow

1. Install the Diffmint VS Code extension.
2. Run any Diffmint command once. If `dm` is not on PATH and `diffmint.cliPath` is still the default, the extension will install a managed CLI automatically.
3. Set `diffmint.webBaseUrl` if you are pointing the extension at a self-hosted or local control plane instead of `https://diffmint.deplio.app`.
4. Click **Sign In** in the extension.
5. Complete browser auth and return to the editor.
6. Run **Review Current Changes** from the command palette.

If your organization manages the CLI separately, set `diffmint.cliPath` to that binary and the
extension will use it instead of the managed fallback.

## What the editor experience should feel like

Once connected, users should be able to:

- launch review from the current file, staged changes, or working tree
- inspect findings without leaving the editor
- jump from results back into the exact file location that needs attention
- stay aligned with the same workspace and policy model used by the CLI

## Expected commands

The extension uses the same flows as the CLI:

- `dm review`
- `dm review --staged`
- `dm review --files ...`
- `dm explain <file>`
- `dm tests <file>`

Read the [VS Code guide](/docs/vscode/guide) for command mapping and result panels.

## Packaging for internal rollout

For internal rollout or release automation, package the extension as a VSIX:

```bash
pnpm package:vscode
```
