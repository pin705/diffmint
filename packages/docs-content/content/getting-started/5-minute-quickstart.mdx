# 5-Minute Quickstart

This is the fastest path from zero to a real Diffmint workflow. By the end of this guide, you
will have authenticated a machine, approved the device session, run a review against real code,
and confirmed that history is visible in the control plane.

Diffmint is local-first by design. The primary experience lives in the CLI and VS Code extension.
The web app is the control plane for workspace setup, policy, billing, history, and audit.

## What you need

- A local checkout of the repository you want to review
- `pnpm` available on your machine
- Browser access for the one-time sign-in and device approval step
- A workspace invitation if your team already operates Diffmint centrally

## 1. Authenticate from the CLI

Install dependencies and start the device login flow:

```bash
pnpm install
dm auth login
```

Your browser opens to the Diffmint sign-in flow. If your account belongs to multiple workspaces,
switch to the correct one, then approve the device session explicitly. That approval step is what
allows the CLI to fetch workspace bootstrap, read synced history, and upload review artifacts.

## 2. Run a real review

Run the review against your current work instead of a demo command:

```bash
dm review --base origin/main
```

What you should expect:

- Diffmint inspects only the files needed for the review
- Provider execution stays local-first unless your workspace sync rules say otherwise
- Findings render directly in the terminal and can be synced into history when enabled

If you prefer to review only staged files, use `dm review --staged`.

## 3. Confirm history and machine health

Once the review finishes, verify that the local machine and workspace connection are both healthy:

```bash
dm history
dm doctor
```

Success usually looks like this:

- `dm history` returns recent review sessions for the active workspace
- `dm doctor` confirms local runtime, auth state, and provider readiness
- The dashboard history view shows the same session metadata after sync

If the control plane is temporarily unavailable, Diffmint keeps sync items locally and flushes them
automatically on the next successful workspace-connected command.

## Admin rollout in parallel

If you are setting up Diffmint for a team instead of just yourself, run this in parallel with the
developer quickstart:

1. Create the workspace.
2. Choose the provider strategy.
3. Publish the first policy version.
4. Confirm billing and quota defaults.
5. Invite a pilot group.
6. Share the CLI and VS Code install guides.
