# CLI Reference

## Auth

Use browser-based device auth when you need control-plane sync, or local BYOK auth when you want
to keep provider credentials on the user machine:

```bash
dm auth login
dm auth login codex
dm auth login antigravity
dm auth login api
dm auth logout
```

`dm auth login` starts a browser-based device flow. After signing in, approve the device page
so the CLI can fetch bootstrap, read synced history, and upload review artifacts with an approved
session. Successful sign-in also registers the local CLI version, platform, and release channel in
the workspace control plane for support and release governance.

The local BYOK variants do not upload provider keys to Diffmint:

- `dm auth login codex` uses `codex login` as the local auth source
- `dm auth login antigravity` keeps Antigravity auth local to the machine
- `dm auth login api` reads a local provider key from `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `QWEN_API_KEY`, or `DASHSCOPE_API_KEY`

## Provider configuration

```bash
dm config set-provider codex
dm config set-model gpt-5-codex
```

Set the provider and model once per machine or workspace context, then validate with `dm doctor`.

## Review workflows

```bash
dm review
dm review --staged
dm review --base origin/main
dm review --files src/a.ts src/b.ts
```

Use the mode that matches the scope of your intent:

- `dm review` for the current working state
- `dm review --staged` for the staged diff only
- `dm review --base origin/main` for branch-level review
- `dm review --files ...` for surgical review on a specific surface

When Codex is installed locally and authenticated, `dm review` uses the Codex runtime and records
the raw provider output as an artifact. When the selected provider is Qwen and the local runtime is
available, Diffmint uses Qwen headless mode. When the selected runtime is missing, unauthenticated,
or disabled with `DIFFMINT_REVIEW_RUNTIME=scaffold`, the CLI falls back to the built-in scaffold
reviewer so local review never blocks.

## Explain and tests

```bash
dm explain src/lib/review.ts
dm tests src/lib/review.ts
```

Use `explain` when a file needs interpretation. Use `tests` when the next step is coverage,
regression protection, or faster local validation.

## History and diagnostics

```bash
dm history
dm history --json
dm history --provider qwen --query auth
dm history --compare latest previous
dm doctor
dm doctor --json
```

These commands are the fastest way to verify that the local environment and control-plane
connection still agree.

`dm history` now defaults to a grouped terminal summary with scope, severity, and completion time.
Use `dm history --json` for scripting or extension integrations. `dm doctor` follows the same rule:
plain text by default for humans, `--json` for tooling.

Use history filters when local context starts to grow:

- `--provider`, `--policy`, and `--source` narrow the session list before rendering
- `--query` searches trace ID, summary, scope, provider, policy, and finding text
- `--limit` trims long local histories without losing the grouped summary format
- `--compare <left> <right>` compares two sessions by trace ID, prefix, `latest`, or `previous`

## Output modes

Use JSON for scripting and Markdown for artifacts:

```bash
dm review --base origin/main --json
dm review --base origin/main --markdown
```

JSON is the right default for automation. Markdown is the right default for saved artifacts,
handoffs, or human review outside the terminal.

## Context conventions

Diffmint review output now follows a compact convention inspired by stronger agent CLIs:

- `Overview` for trace, provider, model, and policy identity
- `Context` for grouped file scope, diff size, and visible file list
- `Severity Summary` to collapse risk counts before the details
- `Findings` for actionable review items only
- `Next Steps` to turn the review into a concrete validation plan

This keeps long diffs readable without flooding the terminal with raw file lists.

## Team review conventions

Diffmint can load workspace review conventions from `.diffmint/review-conventions.json` in the
current repo root:

```json
{
  "promptProfile": "diffmint-team-web-v2",
  "additionalPriorities": [
    "Prefer crisp findings over speculative commentary.",
    "Prioritize auth, policy, and release regressions before style feedback."
  ],
  "reviewNotes": ["Include a concrete validation step for every high-severity finding."],
  "snippetContextLines": 2,
  "maxVisibleFiles": 5,
  "maxFileGroups": 6
}
```

This lets teams tune review prompt profile, code excerpt size, and grouped context limits without
forking the CLI. `dm doctor` reports whether the convention file is loaded, missing, or invalid.
