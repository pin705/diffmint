# CLI Reference

## Auth

Use browser-based device auth to connect a machine to the correct workspace:

```bash
dm auth login
dm auth logout
```

`dm auth login` starts a browser-based device flow. After signing in, approve the device page
so the CLI can fetch bootstrap, read synced history, and upload review artifacts with an approved
session. Successful sign-in also registers the local CLI version, platform, and release channel in
the workspace control plane for support and release governance.

## Provider configuration

```bash
dm config set-provider qwen
```

Set the provider once per machine or workspace context, then validate with `dm doctor`.

## Review workflows

```bash
dm review
dm review --staged
dm review --base origin/main
dm review --files src/a.ts src/b.ts
```

Use the mode that matches the scope of your intent:

- `dm review` for the current working state
- `dm review --staged` for the staged diff only
- `dm review --base origin/main` for branch-level review
- `dm review --files ...` for surgical review on a specific surface

When Qwen Code is installed locally and a compatible provider key is present, `dm review`
uses the Qwen headless runtime and records the raw provider output as an artifact. When Qwen is
missing, unauthenticated, or disabled with `DIFFMINT_REVIEW_RUNTIME=scaffold`, the CLI falls back
to the built-in scaffold reviewer so local review never blocks.

## Explain and tests

```bash
dm explain src/lib/review.ts
dm tests src/lib/review.ts
```

Use `explain` when a file needs interpretation. Use `tests` when the next step is coverage,
regression protection, or faster local validation.

## History and diagnostics

```bash
dm history
dm doctor
```

These commands are the fastest way to verify that the local environment and control-plane
connection still agree.

## Output modes

Use JSON for scripting and Markdown for artifacts:

```bash
dm review --base origin/main --json
dm review --base origin/main --markdown
```

JSON is the right default for automation. Markdown is the right default for saved artifacts,
handoffs, or human review outside the terminal.
