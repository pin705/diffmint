# Local-First Review

Diffmint reads git diffs and local files on the developer machine. The main workflow does not upload the full repository to the cloud.

## What stays local

- Git diff generation
- File reads
- Headless review execution
- Policy injection before review
- Terminal rendering

## What may sync

- Review metadata
- Severity counts
- Selected report artifacts
- Provider and model metadata
- Trace IDs for audit

If the control plane is offline, Diffmint keeps sync payloads in a local queue and flushes them
automatically after the workspace reconnects.

When cloud sync is enabled, Diffmint redacts common secret patterns before upload and omits raw
provider output from synced artifacts by default.

See [Privacy and redaction](/docs/security/privacy-and-redaction) for sync controls and default safeguards.
