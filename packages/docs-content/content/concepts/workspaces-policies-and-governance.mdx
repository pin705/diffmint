# Workspaces, Policies, and Governance

Diffmint is not a generic assistant surface. It is a governed review system where policy and
workspace context are part of the product, not an afterthought.

## What a workspace owns

Each workspace owns:

- provider strategy
- plan and quota
- team rules
- synced review history
- audit events

## What a policy version should capture

Each review can carry:

- policy version ID
- workspace metadata
- provider and model metadata
- trace ID

That bundle makes review output searchable, enforceable, and auditable later.

## Why the metadata matters

Once teams scale, the question is no longer just “what did the model say?” It becomes:

- which workspace approved this behavior?
- which policy version was active at the time?
- which provider and model produced the result?
- can we audit, compare, or replay the decision later?

## Recommended baseline

Start simple:

1. one workspace per operating boundary
2. one published policy version before rollout
3. one agreed provider strategy
4. synced history enabled only when the team understands the privacy boundary
