# Clerk local runtime fallback

- Public routes, docs, and install flows now render locally even when Clerk keys are not configured.
- Protected auth flows still require real Clerk credentials and show a clear fallback notice when missing.
- This keeps local production-style smoke tests from hanging on optional auth setup.
