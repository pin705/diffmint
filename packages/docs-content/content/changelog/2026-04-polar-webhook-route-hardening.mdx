# Polar Webhook Route Hardening

- `/api/polar/webhooks` now rejects requests that do not include the expected signed Polar webhook headers
- oversized webhook deliveries are rejected before they reach the Polar adapter
- every webhook response now includes a request id and `no-store` cache headers for support and traceability
